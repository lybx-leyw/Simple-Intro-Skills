# Contributing

## Report Issues

- **Bug**: skill doesn't work as documented. Include skill name, expected vs actual behavior.
- **Improvement**: dimension weight off? Edge case missing? Open an issue.
- **Feature**: idea for a new skill that fits the trilogy.

## Submit PRs

1. Fork the repo.
2. Make changes in a feature branch.
3. Run a self-assessment on your changes (bonus points for `/Intro`).
4. Open a PR describing what changed and why.

## Add a New Skill

1. Follow `SKILL.md` frontmatter format (`name`, `description`, `runAs`).
2. Place at `skills/<your-skill>/SKILL.md`.
3. Update the README resource table.

## Principles

- Skill definitions must be self-contained — a `SKILL.md` should work standalone.
- Edge cases matter — document them in a table.
- Chinese for skill instructions; English for repo documentation.
